## Why

Web 应用存在大量高频漏洞（OWASP Top 10 范围），逐个修补成本高且容易遗漏。本课设通过构建一个具有真实业务场景的留言管理系统（含普通用户和 admin 两种角色），在其中植入全面的常见 Web 漏洞，并在其上叠加一层通用防护中间件（通防模式），演示"统一拦截"比"逐点修补"更系统的防护思路。

## What Changes

- 构建一个**留言管理系统**作为漏洞载体，分普通用户和 admin 两种角色，含登录、留言发布、用户管理、文件上传/下载、系统工具等功能模块
- 在系统中植入 10 类常见 Web 漏洞：SQL 注入、存储型 XSS、反射型 XSS、CSRF、路径穿越、越权访问（水平+垂直）、不安全文件上传、命令注入、敏感信息泄露、弱口令/暴力破解
- 实现通防安全中间件，在 `before_request` 钩子中统一检测和拦截恶意输入
- 在 `after_request` 钩子中统一注入安全响应头（CSP、X-Frame-Options 等）
- 提供攻击演示脚本，用于前后对比测试
- 输出带拦截日志的防护版应用，与漏洞版形成可视化对比

## Capabilities

### New Capabilities

- `vulnerable-app`: 留言管理系统（漏洞版），含普通用户和 admin 两种角色，覆盖 10 类常见 Web 漏洞
- `defense-middleware`: 通防安全中间件，统一处理输入检测（SQL/XSS/命令注入/路径穿越）、CSRF Token 验证、越权访问控制、文件上传校验、频率限制、安全响应头注入
- `attack-demo`: 攻击演示脚本集合，覆盖全部 10 类漏洞，用于驱动前后对比测试
- `test-comparison`: 自动化对比测试，分别对漏洞版和防护版执行相同攻击，记录结果差异

## Impact

- 运行环境：Python 3.x + Flask，本地运行，使用 SQLite
- 无外部依赖服务，全部本地演示
- 漏洞版和防护版共享同一套业务模板，仅中间件层不同
